1 Project Introduction

Business Problem

During a global health event, decision-makers need a fast, visual way to answer: how many cases exist, where are they concentrated, and is the situation improving or worsening over time?

Project Goal

Build a 3-page interactive Power BI dashboard tracking confirmed cases, deaths, and recoveries across countries and over time.

What Students Will Build

A dashboard with global KPI cards, a country comparison view with a map, and a trend page showing how the numbers changed day by day.

Learning Outcomes

  • Work with a real date column and build a proper Calendar table
  • Practice basic Time Intelligence DAX
  • Build map visuals for geographic data
  • Design 3 clean, interactive dashboard pages

4 Data Inspection

Check In Data View

  • Column Names — clean and readable?
  • Row Count — many rows per country (one per date)
  • Data Type of "Date" column — often imports as Text

Expected Observations

  • Confirmed, Deaths usually numeric already
  • Recovered often has nulls, especially for recent dates
  • Some country names may have extra spaces or inconsistent spelling
Report View → Left Panel → Data View (table icon) → Scroll Through Columns
⚠ Blank Values
If Recovered has many blanks, decide as a class: replace with 0 (assume not yet recorded) rather than leaving it null, so SUM() doesn't silently skip rows.

5 Power Query (Data Cleaning)

Step 1: Remove Duplicates

Home → Transform Data → Select "Country" + "Date" Columns (Ctrl+Click both) → Right-Click → Remove Duplicates

Why: no country should have two rows for the same date.

Step 2: Handle Null Values

Select "Recovered" Column → Right-Click → Replace Values → null → 0

Why: nulls break SUM() and AVERAGE() silently — replacing with 0 keeps totals accurate.

Step 3: Trim & Clean Country Names

Select "Country" Column → Transform Tab → Format → Trim → Format → Clean

Why: hidden spaces make "India" and "India " look like two different countries in slicers and maps.

Step 4: Rename Columns

Double-Click Column Header → Type New Name → Enter

Example: keep names simple and consistent: Country, Date, Confirmed, Deaths, Recovered.

Step 5: Close & Apply

Home Tab → Close & Apply
💡 Hint
Clean the Country column before building the map visual — misspelled or inconsistent country names will not plot correctly on a map.

6 Data Type Conversion

This must happen before any DAX — especially the Date column, since Time Intelligence functions require a true Date type.

Select "Date" Column → Column Tools Tab → Data Type → Date
Select "Confirmed", "Deaths", "Recovered" → Data Type → Whole Number
💡 Hint
If Date stays as Text, your Calendar Table relationship will fail and Time Intelligence DAX (like month-over-month growth) will not work at all.
⚠ Wrong Data Type
If the Date column shows as a number of days instead of an actual date after conversion, check your regional date format under File → Options → Regional Settings, and re-import if needed.

7 Data Modeling

This dataset has one main table (COVID Data), and we will add a second table: a Calendar Table. This is a classic Star Schema pattern.

Explain to Students

  • Fact Table: our COVID Data table (holds the numbers: Confirmed, Deaths, Recovered)
  • Dimension Table: the Calendar table (holds Date-related descriptive fields: Year, Month, Quarter)
  • Relationship: One-to-Many from Calendar[Date] to COVID Data[Date]
Model View → Drag "Date" from Calendar Table → Drop on "Date" in COVID Data Table → Confirm Cardinality: One-to-Many → Cross Filter Direction: Single → OK
⚠ Missing Relationship
If you filter by Year on a slicer and nothing changes, the relationship between Calendar and COVID Data likely isn't active — check Model View for a solid connecting line.

8 Calendar Table

Modeling Tab → New Table → Paste DAX Below
Calendar = 
ADDCOLUMNS(
    CALENDAR(MIN('COVID Data'[Date]), MAX('COVID Data'[Date])),
    "Year", YEAR([Date]),
    "Month Number", MONTH([Date]),
    "Month Name", FORMAT([Date], "MMM"),
    "Quarter", "Q" & FORMAT([Date], "Q")
)

Explanation: CALENDAR() generates one row per date between the earliest and latest date in the COVID Data table. ADDCOLUMNS then adds readable Year, Month Name, and Quarter fields for slicers and axis labels.

💡 Hint
Always build the Calendar table from MIN()/MAX() of your actual data's date range — never hardcode start/end dates, so it stays accurate if the dataset updates.

9 Create Measures

Why Measures over Calculated Columns: measures recalculate live as users apply slicers and filters, keeping every number on the dashboard accurate and interactive.

Modeling Tab → New Measure → Write DAX → Press Enter

1. Total Confirmed

Total Confirmed = SUM('COVID Data'[Confirmed])

2. Total Deaths

Total Deaths = SUM('COVID Data'[Deaths])

3. Total Recovered

Total Recovered = SUM('COVID Data'[Recovered])

4. Mortality Rate %

Mortality Rate % = DIVIDE([Total Deaths], [Total Confirmed], 0)

Purpose: the key risk indicator — what share of confirmed cases resulted in death.

5. Recovery Rate %

Recovery Rate % = DIVIDE([Total Recovered], [Total Confirmed], 0)

6. Active Cases

Active Cases = [Total Confirmed] - [Total Deaths] - [Total Recovered]

Purpose: cases still ongoing — neither recovered nor resulting in death.

💡 Hint
Rename measures with business-friendly names and keep percentage measures formatted as % in the Format pane (Measure Tools → Format → Percentage) so they display correctly on cards.

10 Dashboard Building

We will build 3 pages. Follow this visual-by-visual approach for every chart: drag field → format pane → polish.

Page 1 — Global Overview

Executive Dashboard
Card Total Confirmed
Visualizations Pane → Card → Drag "Total Confirmed" → Format Pane → Category Label OFF → Increase Font Size → Change Background → Apply Shadow
Card Total Deaths, Total Recovered, Mortality Rate %
Repeat Card steps for each measure → Arrange as a KPI row at the top of the page
Line Chart Confirmed Cases Over Time
Visualizations Pane → Line Chart → X-Axis: Calendar[Date] → Y-Axis: Total Confirmed
Why: a line chart is the clearest way to show growth over time.
Map Confirmed Cases by Country
Visualizations Pane → Map (or Filled Map) → Location: Country → Size/Color Saturation: Total Confirmed
Why: geographic spread is instantly understood visually, better than a table of country names.

Scatter Chart → X-Axis: Total Confirmed → Y-Axis: Total Deaths → Legend: Country
Why: reveals which countries have disproportionately high deaths relative to case count.
Matrix Country Summary Table

Page 3 — Trend Analysis

Time Trend Dashboard
Line Chart Confirmed, Deaths & Recovered Over Time
Line Chart → X-Axis: Calendar[Date] → Y-Axis: Total Confirmed, Total Deaths, Total Recovered (all 3 as separate lines)
Why: comparing all 3 trend lines together shows the overall trajectory of the situation.
Area Chart Active Cases Over Time
Area Chart → X-Axis: Calendar[Date] → Y-Axis: Active Cases

Clustered Column Chart → Axis: Calendar[Month Name] → Values: Total Confirmed

Visualizations Pane → Slicer → Field: Calendar[Year] → Place at top of page
Why: lets viewers narrow the trend view to a specific year without editing the report.
