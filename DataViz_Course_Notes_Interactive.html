Part 1

Matplotlib — the foundation

Matplotlib is the low-level, do-anything plotting library that almost every other Python visualization tool (including Seaborn) is built on top of. It gives you total control — which also means you write a bit more code. Master this first and everything after it will feel easy.

01What is Matplotlib?concept›

Simple explanation

Matplotlib is a Python library for drawing charts and graphs. Think of it as a digital pen and graph paper: you tell it exactly where to draw lines, bars, or dots, and it draws them for you. It's called "Matplotlib" because it was inspired by MATLAB's plotting style.

Why it's used

  • To see your data — patterns are much easier to spot in a picture than in a spreadsheet.
  • To create clean, publication-quality figures for reports and papers.
  • Because it's the foundation other libraries (Seaborn, Pandas' .plot()) are built on — learning it first makes everything else make sense.

The anatomy of a Matplotlib figure

Before writing any code, you need to know the four building blocks:

One Figure can hold one or many Axes. Each Axes has its own X/Y Axis (scale + ticks) and plot elements.
  1. Figure → the whole window/page that contains everything.
  2. Axes → the actual rectangle where your data gets drawn (confusingly, this is singular "an Axes", not related to "axis").
  3. Axis → the x and y number lines — their scale, ticks, and limits.
  4. Plot elements → titles, labels, legends, lines, bars, markers.
Important notePeople mix up "Axes" (the plot area) and "Axis" (the x/y number line) constantly — even experienced coders. Axes = the container. Axis = the ruler on its edge.

Typical workflow

python
import matplotlib.pyplot as plt

That one line is how every Matplotlib session starts. plt is just a nickname (alias) for the module so you don't have to type matplotlib.pyplot every time.

Common beginner mistakes

  • Forgetting the import line and getting a NameError: name 'plt' is not defined.
  • Typing import matplotlib alone and then trying to call plt.plot() — you specifically need the .pyplot submodule.
💡
Key takeaway: Matplotlib = Figure (the page) + Axes (the plotting area) + Axis (the rulers) + plot elements. Everything you'll learn is just different ways of creating and arranging these pieces.
02Your First Line Plotcode›

Simple explanation

The simplest thing Matplotlib can do is connect a set of points with a line — a line plot. You give it two lists of numbers (x-values and y-values), and it draws a line through them.

Why it's used

Line plots are the go-to choice for showing a trend — how something changes over time or over an ordered sequence (sales by month, temperature by day, stock price over the year).

Syntax

plt.plot(x, y) — pass an x-list and a y-list of the same length; Matplotlib pairs them up point by point and draws lines between them.

python — basic line plot
import matplotlib.pyplot as plt

# Two simple arrays
x = [1, 2, 3, 4, 5]
y = [10, 20, 25, 30, 40]

# Create a basic line plot
plt.plot(x, y)

# Add labels and title
plt.xlabel("X values")
plt.ylabel("Y values")
plt.title("Basic Line Plot with Two Arrays")

# Show the plot
plt.show()

Line-by-line walkthrough

import matplotlib.pyplot as pltloads the plotting toolbox and gives it the short name plt.
x = [1,2,3,4,5]the horizontal positions of each point.
y = [10,20,25,30,40]the matching heights — point 1 is (1,10), point 2 is (2,20), and so on.
plt.plot(x, y)draws the line connecting those 5 points, in memory (not on screen yet).
plt.xlabel/ylabelnames the axes so the reader knows what the numbers mean.
plt.title(...)gives the whole chart a heading.
plt.show()actually renders/opens the finished chart. Nothing appears on screen until this line runs.
Common mistakeForgetting plt.show() in a plain .py script — the chart is built but never displayed. (In Jupyter notebooks, plots often auto-display anyway, but it's still good habit to include it.)
Common mistakex and y must be the same length. A mismatched list length raises a ValueError.
Quick check

What happens if you remove plt.show() from a standalone Python script?

💡
Key takeaway: plt.plot(x, y) draws the line, and plt.show() is what makes it appear. Always label your axes and title — an unlabeled chart tells no story.
03The Canvasconcept›

Simple explanation

A canvas is the blank sheet of paper your plots get drawn on. It's not a special Matplotlib term you'll see in the docs often, but it's the easiest way to think about plt.figure() — before you draw anything, you set up the blank page first, then draw one or several plots on top of it.

Why it's used

You create a canvas whenever you want to control the size of the output, or when you want to put multiple lines/plots together on the same page instead of Matplotlib guessing a default size for you.

Syntax

plt.figure(figsize=(width, height)) — width and height are in inches. You create the canvas before calling plt.plot().

python — one canvas, two lines
import matplotlib.pyplot as plt

# Create one canvas and draw multiple plots on it
x = [1, 2, 3, 4, 5]
y1 = [1, 4, 9, 16, 25]
y2 = [1, 2, 3, 4, 5]

plt.figure(figsize=(6,4))   # <-- create the canvas

plt.plot(x, y1, label="Squares", color="blue")
plt.plot(x, y2, label="Line", color="red")

plt.title("Multiple Plots on the Same Canvas")
plt.xlabel("X")
plt.ylabel("Y")
plt.legend()
plt.grid(True)

plt.show()

Line-by-line walkthrough

plt.figure(figsize=(6,4))opens a blank canvas 6 inches wide, 4 inches tall.
plt.plot(x, y1, label=..., color=...)draws the first line in blue and remembers its name ("Squares") for later.
plt.plot(x, y2, ...)draws a second line on the same canvas, in red — Matplotlib doesn't erase what came before.
plt.legend()shows a small box mapping each color to its label.
plt.grid(True)adds light gridlines to make reading values easier.
TipBecause plt.plot() keeps drawing on the same "current" canvas until you call plt.show() or start a new plt.figure(), you can layer as many plots as you like on top of each other.
Common mistakeCalling plt.figure() a second time by accident starts a brand-new blank canvas — any plot commands after that will NOT land on your first chart.
💡
Key takeaway: Think "canvas first, then draw." plt.figure() sets up the blank page and its size; everything you plot afterward lands on that same page until you open a new one.
04Subplots with plt.subplot()code›

Simple explanation

plt.subplot() splits one figure into a grid of smaller plotting areas, so you can show several charts side by side instead of one giant chart.

Why it's used

Great for comparisons — e.g., showing "before vs after," or three different chart types describing the same data, without opening three separate windows.

Syntax

plt.subplot(nrows, ncols, index)

  • nrows — how many rows the grid has
  • ncols — how many columns the grid has
  • index — which cell to draw in next, counted left→right, top→bottom, starting at 1 (not 0!)
A 2×2 grid — cells are numbered 1→4, left to right then top to bottom. Our code below fills cells 1 and 4.
python — 2×2 grid
import matplotlib.pyplot as plt

x = [1, 2, 3, 4, 5]
y1 = [2, 4, 6, 8, 10]
y2 = [1, 4, 9, 16, 25]

# First plot in position 1 of a 2x2 grid
plt.subplot(2, 2, 1)
plt.plot(x, y1, 'r--')
plt.title("Plot 1")

# Second plot in position 4 of a 2x2 grid
plt.subplot(2, 2, 4)
plt.plot(x, y2, 'g*')
plt.title("Plot 2")

plt.tight_layout()
plt.show()

Line-by-line walkthrough

plt.subplot(2,2,1)"the next plot goes in a 2×2 grid, cell #1 (top-left)."
plt.plot(x, y1, 'r--')the shorthand string 'r--' means red, dashed line.
plt.subplot(2,2,4)now switch to cell #4 (bottom-right) of the same 2×2 grid.
plt.plot(x, y2, 'g*')'g*' means green stars as markers, with no connecting line.
plt.tight_layout()automatically adds spacing so titles/labels don't overlap between subplots.
Common mistakeForgetting that indexing starts at 1, not 0 — plt.subplot(2,2,0) will raise an error.
TipAlways finish a multi-subplot figure with plt.tight_layout() — it's the single easiest fix for "my titles are overlapping" complaints.
Quick check

In a 3×1 grid, which call draws in the very last (bottom) cell?

💡
Key takeaway: plt.subplot(rows, cols, index) — index starts at 1 and moves left→right, top→bottom. Perfect for quick side-by-side comparisons.
05Object-Oriented Interfaceconcept›

Simple explanation

So far we've been telling plt what to do directly ("plt, draw a line"). The Object-Oriented (OO) style instead creates actual named objects for the Figure and the Axes, and you call methods directly on them (ax.plot(...) instead of plt.plot(...)).

Why it's used

The plt shortcuts always act on "whatever the current figure/axes is" — which gets confusing once you have several plots. The OO style gives you a direct handle on exactly the Figure and Axes you want, which is why it's preferred for anything more complex than a single quick plot.

Syntax

  1. Create a Figure (the canvas): fig = plt.figure(figsize=(6,4))
  2. Add Axes to it: ax = fig.add_axes([left, bottom, width, height]) — all four numbers are fractions from 0 to 1 of the figure's size.
  3. Plot and customize using ax. methods instead of plt. methods.
python — OO style plot
import matplotlib.pyplot as plt

# Sample data
x = [1, 2, 3, 4, 5]
y = [2, 4, 6, 8, 10]

# Step 1: Create a Figure (canvas)
fig = plt.figure(figsize=(6,4))

# Step 2: Add Axes (x0, y0, width, height) in 0-1 range
ax = fig.add_axes([0.1, 0.1, 0.8, 0.8])

# Step 3: Plot data on the Axes
ax.plot(x, y, color="purple", marker="o", linestyle="-")

# Step 4: Customize
ax.set_title("Object-Oriented Plot")
ax.set_xlabel("X-axis")
ax.set_ylabel("Y-axis")
ax.grid(True)

plt.show()

Line-by-line walkthrough

fig = plt.figure(figsize=(6,4))creates the blank canvas and saves a reference to it in the variable fig.
ax = fig.add_axes([0.1,0.1,0.8,0.8])adds one plotting rectangle starting 10% from the left, 10% from the bottom, taking up 80% of the width and height.
ax.plot(x, y, ...)draws directly onto that specific Axes object — no ambiguity about "which plot is current."
ax.set_title / set_xlabel / set_ylabelnote the set_ prefix — this is the OO-style naming convention (vs. plt.title() in the quick style).
NoteQuick style (plt.plot) and OO style (ax.plot) do the same underlying thing. Quick style is faster to type for one simple plot; OO style is clearer and safer once things get more complex.
Common mistakeMixing up plt.xlabel() (quick style) with ax.xlabel() — the OO method is actually named ax.set_xlabel(), not ax.xlabel().
💡
Key takeaway: plt.___() = quick and implicit. fig + ax.set____() = explicit and safe for anything with multiple plots. Learn to read both — most real-world code mixes them.
06Multiple / Inset Axescode›

Simple explanation

Because fig.add_axes() just needs four position numbers, you can add a second, smaller Axes anywhere on the same Figure — even floating on top of the first one, like a zoomed-in inset.

Why it's used

Useful for showing a "zoomed-in" detail of part of your chart, or two related but differently-scaled plots that should live on the same page.

Two Axes on one Figure — a big main plot and a small inset plot placed on top of it.
python — two axes, one figure
fig = plt.figure()
axes1 = fig.add_axes([0.1,0.1,0.8,0.8])
plt.title("hello")
axes2 = fig.add_axes([0.2,0.5,0.3,0.3])
plt.title("world")
axes1.plot(x,y)
axes2.plot(y,x)

Line-by-line walkthrough

axes1 = fig.add_axes([0.1,0.1,0.8,0.8])the big main plot, covering most of the figure.
axes2 = fig.add_axes([0.2,0.5,0.3,0.3])a small plot starting 20% from the left, 50% up from the bottom, only 30% wide and tall — it sits on top of axes1 like an inset.
axes1.plot(x,y) / axes2.plot(y,x)each Axes gets its own independent plot — notice axes2 even plots (y,x), the reverse of axes1.
Best practiceGive each Axes object a clear variable name (axes1, axes2, or better — ax_main, ax_inset) so it's obvious later which plot you're customizing.
Common mistakeCalling plt.title("hello") right after creating axes1 actually sets the title of whichever Axes is "current" at that moment — it's clearer and safer to use axes1.set_title("hello") directly.
💡
Key takeaway: One Figure can hold as many Axes as you want, placed anywhere using fractional coordinates — this is how "zoomed-in inset" charts are built.
07Figure Size, Aspect Ratio & DPIcode›

Simple explanation

  • Figure size — the physical width × height of the canvas, measured in inches.
  • Aspect ratio — the relationship between that width and height (a 6×4 figure is wider than tall).
  • DPI (Dots Per Inch) — how many pixels are packed into each inch. Higher DPI = sharper, more detailed image (and a bigger file).

Why it's used

You control these so your chart looks right wherever it ends up — readable on a slide, crisp in a printed report, or appropriately sized inside a dashboard.

Syntax

plt.figure(figsize=(width, height), dpi=value)

Same physical size (8×4 inches), but doubling DPI doubles the pixel resolution — sharper on screen, bigger file.
python — custom size & DPI
import matplotlib.pyplot as plt

x = [1, 2, 3, 4, 5]
y = [2, 4, 6, 8, 10]

# Create a figure with custom size & DPI
plt.figure(figsize=(8, 4), dpi=120)   # width=8in, height=4in, 120 pixels/inch

plt.plot(x, y, color="teal", marker="o")
plt.title("Custom Figure Size & DPI")
plt.xlabel("X-axis")
plt.ylabel("Y-axis")
plt.grid(True)

plt.show()

Line-by-line walkthrough

figsize=(8,4)the canvas is 8 inches wide and 4 inches tall — a 2:1 aspect ratio.
dpi=120each inch contains 120 pixels, so the final image is 960×480 pixels.
TipFor sharing on the web, 100–150 DPI is plenty. For printing, aim for 300 DPI.
Common mistakeSetting a huge DPI "just to be safe" — this bloats file size and can make Jupyter notebooks slow, without any visible benefit on a normal screen.
💡
Key takeaway: figsize controls physical dimensions, dpi controls sharpness/resolution. Together they decide the final pixel dimensions of your chart.
08Legends, Labels & Titlescode›

Simple explanation

These are the "words" of a chart — without them, even a perfect chart is meaningless. A title says what the chart is about, axis labels say what each axis measures, and a legend tells you which color/line belongs to which category.

Why it's used

A chart with unlabeled axes forces the reader to guess. Good labeling is what turns "a pretty picture" into "a piece of communication."

Syntax

  • plt.title("...")
  • plt.xlabel("...") / plt.ylabel("...")
  • plt.legend(loc="upper left") — loc controls where the legend box sits.
python — legend, labels, title
import matplotlib.pyplot as plt

# Sample data
x = [1, 2, 3, 4, 5]
y1 = [1, 4, 9, 16, 25]
y2 = [1, 2, 3, 4, 5]

plt.figure(figsize=(7,4))

# Plot first line
plt.plot(x, y1, color="blue", marker="o", label="Squares")

# Plot second line
plt.plot(x, y2, color="red", marker="s", label="Line")

# Add labels and title
plt.xlabel("X-axis")
plt.ylabel("Y-axis")
plt.title("Example: Legends, Labels & Title")

# Show legend
plt.legend(loc="upper left")

plt.grid(True)
plt.show()

Line-by-line walkthrough

label="Squares"this string is what will show up in the legend for this line — it does nothing without plt.legend().
plt.legend(loc="upper left")draws the legend box in the upper-left corner of the chart.
Common mistakeAdding label= to your plot() calls but forgetting to call plt.legend() — the labels are stored but never shown until you ask for the legend.
TipTry loc="best" and Matplotlib will automatically pick a spot that overlaps your data the least.
💡
Key takeaway: Title says "what," axis labels say "in what units," legend says "which line is which." Always include all three on any chart you share with others.
09Line Styles, Colors, Widths & Markerscode›

Simple explanation

Matplotlib lets you customize almost everything about how a line looks: its color, thickness, dash pattern, and the little symbols (markers) placed at each data point.

Why it's used

When you have several lines on one chart, different styles help the reader tell them apart — especially important in black-and-white print or for colorblind-friendly design.

Syntax

  • color= or c= — the line's color
  • linewidth= or lw= — thickness
  • linestyle= or ls= — '-' solid, '--' dashed, '-.' dash-dot, ':' dotted
  • marker= — symbol at each point ('o' circle, 's' square, '*' star, 'x' x-mark), plus markersize=, markerfacecolor=, markeredgecolor=
python — all the styling options
import matplotlib.pyplot as plt

# Sample data
x = [0, 1, 2, 3, 4, 5]

fig, ax = plt.subplots(figsize=(12, 6))

# Different line widths (all in orange)
ax.plot(x, [v + 1 for v in x], color="orange", linewidth=0.5, label="width=0.5")
ax.plot(x, [v + 2 for v in x], color="orange", linewidth=1.5, label="width=1.5")
ax.plot(x, [v + 3 for v in x], color="orange", linewidth=3, label="width=3")

# Line styles with constant width
ax.plot(x, [v + 4 for v in x], color="green", lw=2, linestyle='-')    # solid
ax.plot(x, [v + 5 for v in x], color="green", lw=2, linestyle='--')   # dashed
ax.plot(x, [v + 6 for v in x], color="green", lw=2, linestyle='-.')   # dash-dot
ax.plot(x, [v + 7 for v in x], color="green", lw=2, linestyle=':')    # dotted

# Custom dash pattern
line, = ax.plot(x, [v + 8 for v in x], color="black", lw=2)
line.set_dashes([10, 5, 2, 5])  # dash, space, dash, space...

# Marker styles with a blue line
ax.plot(x, [v + 9 for v in x], color="blue", lw=2, ls='-', marker='o', label="circle")
ax.plot(x, [v + 10 for v in x], color="blue", lw=2, ls='--', marker='s', label="square")
ax.plot(x, [v + 11 for v in x], color="blue", lw=2, ls='-', marker='*', label="star")
ax.plot(x, [v + 12 for v in x], color="blue", lw=2, ls=':', marker='x', label="x-mark")

# Playing with marker size & colors
ax.plot(x, [v + 13 for v in x], color="purple", lw=1, ls='-', marker='o', markersize=3)
ax.plot(x, [v + 14 for v in x], color="purple", lw=1, ls='-', marker='o', markersize=6)
ax.plot(x, [v + 15 for v in x], color="purple", lw=1, ls='-', marker='o',
        markersize=8, markerfacecolor="yellow", markeredgewidth=2, markeredgecolor="red")

# Final touches
ax.set_title("Line Styles, Colors, Widths & Markers", fontsize=14)
ax.set_xlabel("X-axis")
ax.set_ylabel("Y-axis")
ax.grid(True)
ax.legend(loc="upper left", fontsize=9)

plt.show()

Line-by-line walkthrough

[v + 1 for v in x]a list comprehension — shorthand for "add 1 to every number in x." Used here just to stack lines vertically so they don't overlap.
line, = ax.plot(...)ax.plot() actually returns a list of line objects (even for one line) — the comma unpacks that single item into line.
line.set_dashes([10,5,2,5])defines a custom dash rhythm in pixels: 10 on, 5 off, 2 on, 5 off, repeating.
markerfacecolor / markeredgecolorlets the inside and the outline of a marker be different colors — here a yellow circle with a red outline.
Common mistakeWriting fig, ax = plt.subplots() and then still calling plt.plot() instead of ax.plot() — this can silently draw onto the wrong Axes once you have more than one.
Best practiceDon't overload a single chart with every style at once (as our demo intentionally does!). In real work, use 2–4 distinct styles maximum so the chart stays readable.
💡
Key takeaway: Color, width, dash style, and markers are your toolbox for making multiple lines distinguishable — use them with restraint, not all at once.
10Setting Plot Range (Axis Limits)code›

Simple explanation

By default, Matplotlib automatically zooms to fit all your data. Sometimes you want to control that zoom yourself — showing extra empty space around the data, or cropping in tighter.

Why it's used

Useful for keeping consistent axis ranges across several charts (so they're visually comparable), or to zoom into a specific region of interest.

Syntax

  • Quick style: plt.xlim(min, max) / plt.ylim(min, max)
  • OO style: ax.set_xlim(min, max) / ax.set_ylim(min, max)
python — custom plot range
import matplotlib.pyplot as plt

x = [0, 1, 2, 3, 4, 5]
y = [0, 1, 4, 9, 16, 25]

fig, ax = plt.subplots(figsize=(7, 4))

ax.plot(x, y, color="teal", marker="o", linewidth=2)

# Set custom plot range
ax.set_xlim(-1, 6)      # x-axis range
ax.set_ylim(-5, 30)     # y-axis range

ax.set_title("Custom Plot Range")
ax.set_xlabel("X-axis")
ax.set_ylabel("Y-axis")
ax.grid(True)

plt.show()

Line-by-line walkthrough

ax.set_xlim(-1, 6)forces the visible x-range from -1 to 6, even though the data only goes from 0 to 5 — this adds breathing room on both sides.
ax.set_ylim(-5, 30)same idea for the y-axis, giving padding above and below the data (0 to 25).
Common mistakeSetting limits that are tighter than your data range by accident — this silently crops out real data points from the visible chart without any error or warning.
💡
Key takeaway: Use xlim/ylim (or set_xlim/set_ylim) to take manual control of zoom and framing instead of relying on Matplotlib's auto-zoom.
11Scatter, Histogram & Boxplotcode›

Simple explanation

Three special-purpose plot types, each answering a different question:

  • Scatter plot (plt.scatter) — shows individual points to reveal a relationship between two variables.
  • Histogram (plt.hist) — groups numbers into "bins" to show the shape/spread of one variable's distribution.
  • Boxplot (plt.boxplot) — summarizes a distribution's median, quartiles, and outliers in one compact shape.
python — scatter, hist, box side by side
import matplotlib.pyplot as plt
import numpy as np

# Data
x = [1, 2, 3, 4, 5]
y = [5, 7, 4, 6, 8]
nums = np.random.randn(100)

plt.figure(figsize=(12,4))

# 1. Scatter
plt.subplot(1,3,1)
plt.scatter(x, y, color="red")
plt.title("Scatter")

# 2. Histogram
plt.subplot(1,3,2)
plt.hist(nums, bins=10, color="skyblue")
plt.title("Histogram")

# 3. Boxplot
plt.subplot(1,3,3)
plt.boxplot(nums)
plt.title("Boxplot")

plt.tight_layout()
plt.show()

Line-by-line walkthrough

nums = np.random.randn(100)generates 100 random numbers from a "normal" (bell-curve) distribution, using NumPy.
plt.scatter(x, y, color="red")plots each (x,y) pair as an individual red dot — no connecting line.
plt.hist(nums, bins=10, ...)splits the 100 numbers into 10 equal-width buckets ("bins") and draws a bar for how many numbers fall in each.
plt.boxplot(nums)draws the classic "box and whiskers": the box spans the middle 50% of values, the line inside is the median, and the whiskers/dots show the range and outliers.
Use scatter when...you want to see if two numeric variables move together (e.g., height vs. weight).
Avoid scatter when...you only have one variable, or the data is categorical — there's nothing to "relate."
Use a histogram when...you want to understand the shape of a single variable's spread (e.g., are most exam scores clustered around 70?).
Avoid a histogram when...you need to compare exact groups/categories — a bar chart does that better.
Use a boxplot when...you want a compact summary of spread and outliers, especially to compare several groups at once.
Avoid a boxplot when...your audience isn't familiar with quartiles — it can look intimidating without context.
Real-world exampleA retailer might scatter-plot "ad spend vs. daily sales" to spot a relationship, histogram "order sizes" to see typical basket value, and boxplot "delivery time by region" to compare which region has the most delays and outliers.
Common mistakeChoosing too many/too few histogram bins — too few hides the shape, too many makes it look noisy and spiky. Try a few values and see what tells the clearest story.
Quick check

You want to compare delivery times across 5 warehouses and spot outliers in each. Which chart fits best?

Best practiceMatplotlib can make all three of these, but as you'll see next, Seaborn makes them prettier and shorter to write — especially once "hue" grouping and Pandas DataFrames get involved.
💡
Key takeaway: Scatter = relationship between two variables. Histogram = shape of one variable. Boxplot = summary + outliers, great for comparing groups. Same data, three different questions answered.
Part 2

Seaborn — visualization, simplified

Seaborn sits on top of Matplotlib and does two big favors for you: modern-looking defaults out of the box, and native support for Pandas DataFrames — so instead of pulling out lists by hand, you just point Seaborn at your columns.

12What is Seaborn?concept›

Simple explanation

Seaborn is a high-level plotting library built on Matplotlib. "High-level" means it hides a lot of the manual setup Matplotlib requires and gives you attractive charts in far fewer lines of code.

Why it's used

  • Beautiful, modern default styles — no manual color/theme tweaking needed.
  • Much more concise syntax for common statistical charts.
  • Deep integration with Pandas DataFrames — you can just say "plot column A against column B."

What are "relational" plots?

Relational plots show the relationship between two continuous (numeric) variables. Seaborn's two main relational functions are scatterplot() and lineplot() — both belong to what Seaborn calls the "relational" family of plots.

16barplot() vs countplot()code›

Simple explanation

Categorical plots visualize data where at least one variable is a category (day of week, gender, class) rather than a plain number.

  • barplot() — shows the average value of a numeric column for each category, with error bars.
  • countplot() — simply counts how many rows fall into each category (no numeric column needed at all).

Why it's used

Comparing categories is one of the most common visualization tasks: average sales by region, number of customers by plan type, etc.

Syntax

  • sns.barplot(data=df, x="category_col", y="numeric_col")
  • sns.countplot(data=df, x="category_col") — note: no y needed.
python — barplot vs countplot
import seaborn as sns
import matplotlib.pyplot as plt

tips = sns.load_dataset("tips")

plt.figure(figsize=(10,4))

plt.subplot(1,2,1)
sns.barplot(data=tips, x="day", y="total_bill")
plt.title("Average Bill by Day (barplot)")

plt.subplot(1,2,2)
sns.countplot(data=tips, x="day")
plt.title("Number of Visits by Day (countplot)")

plt.tight_layout()
plt.show()

Line-by-line walkthrough

sns.barplot(x="day", y="total_bill")for each day, computes the average total_bill and draws a bar to that height, with a small vertical line showing the confidence interval.
sns.countplot(x="day")for each day, counts how many rows exist — this answers "how many visits happened," not "how much was spent."
Use barplot when...you want to compare an average/summary number across groups (avg. tip by day).
Use countplot when...you just want to know how many observations fall in each category (how many customers visited on each day).
Common mistakePassing a y column to countplot() — it doesn't take one; it always counts rows. If you need averages of a numeric column, use barplot() instead.
Quick check

💡
Key takeaway: barplot = "what's the average value per group?" countplot = "how many rows are in each group?" Different questions, different functions.
17boxplot() & violinplot()code›

Simple explanation

Both show the spread of a numeric variable within each category:

  • boxplot() — median, quartiles (the "box"), and outliers as individual dots.
  • violinplot() — everything a boxplot shows, plus the full shape of the distribution, drawn as a mirrored, smoothed silhouette.

Why it's used

Great for comparing not just averages, but the full spread of values across groups — e.g., is the tip amount consistent on Fridays, or wildly variable?

Syntax

sns.boxplot(data=df, x="category", y="numeric") and sns.violinplot(data=df, x="category", y="numeric") — nearly identical syntax, just swap the function name.

python — box vs violin
import seaborn as sns
import matplotlib.pyplot as plt

tips = sns.load_dataset("tips")

plt.figure(figsize=(10,4))

plt.subplot(1,2,1)
sns.boxplot(data=tips, x="day", y="total_bill")
plt.title("Boxplot")

plt.subplot(1,2,2)
sns.violinplot(data=tips, x="day", y="total_bill")
plt.title("Violinplot")

plt.tight_layout()
plt.show()
A violin plot is a boxplot's shape "inflated" to show the full density curve of the data.
TipIf your audience is unfamiliar with statistics, a boxplot is usually easier to explain quickly; save violinplot for audiences comfortable reading distribution shapes.
Common mistakeUsing a violinplot with very little data (say, under ~20 points per group) — the smoothed shape can imply more data richness than actually exists, misleading readers.
💡
Key takeaway: boxplot = quick summary of spread + outliers. violinplot = boxplot + the actual distribution shape. Use violin only when you have enough data to make the shape meaningful.
18stripplot() & swarmplot()code›

Simple explanation

Both plot every single data point as a dot, grouped by category — no summarizing or averaging.

  • stripplot() — scatters points along the category axis, with a little random jitter so they don't all stack in one vertical line.
  • swarmplot() — like stripplot, but intelligently spreads overlapping points sideways so none of them hide behind each other.

Why it's used

Perfect for small-to-medium datasets where you want to see literally every observation, not just a summary shape.

Syntax

sns.stripplot(data=df, x="category", y="numeric") / sns.swarmplot(data=df, x="category", y="numeric")

python — strip vs swarm
import seaborn as sns
import matplotlib.pyplot as plt

tips = sns.load_dataset("tips")

plt.figure(figsize=(10,4))

plt.subplot(1,2,1)
sns.stripplot(data=tips, x="day", y="total_bill", jitter=True)
plt.title("Stripplot")

plt.subplot(1,2,2)
sns.swarmplot(data=tips, x="day", y="total_bill")
plt.title("Swarmplot")

plt.tight_layout()
plt.show()

Line-by-line walkthrough

jitter=Truenudges each point a small random amount left/right so overlapping points are still visible (this is stripplot's default anyway).
sns.swarmplot(...)no jitter parameter needed — it calculates exact non-overlapping positions instead of random ones.
Common mistakeUsing swarmplot() on a very large dataset (thousands of points) — it gets slow and cramped. Beyond a few hundred points, prefer violinplot() or boxplot().
TipCombine a boxplot or violinplot with a stripplot/swarmplot on top (using the same Axes) to get both the summary shape AND the raw data points in one chart.
💡
Key takeaway: Both show every raw point. stripplot = fast + jittered (works at any scale). swarmplot = precise, non-overlapping, but best for smaller datasets.

21distplot() (Legacy)concept›

Simple explanation

Why it matters

Important warningdistplot() is deprecated — do not use it in new code. Use histplot() for bars (with optional KDE overlay), or kdeplot() for a curve-only view.
